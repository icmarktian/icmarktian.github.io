    一、内嵌函数

    1.1 $rose

    信号/表达式的LSB跳变为1，返回真

    1.2 $fell

    信号/表达式的LSB跳变为0，返回真

    1.3 $stable

    信号/表达式的值保持不变，返回真

    1.4 $past

    若(c&&d)为真，2个周期前(a&&b)必须为真。默认1个周期。

    @(posedge clk)  (c&&d) |-> ($past ( (a&&b) , 2 ) == 1);

    例1：32bit的data bus，需要检测它在dAck信号为1时，全部bit位跳变为0。

    //$fell只能检测LSB的跳变，不能得到所有信号都变为0的结论
aAck |-> $fell(dBus);

//用$past：dBus在dAck信号前一个周期不为零，现在全为0了
dAck |-> ($past(dBus)!=32'b0) && (dBus==32'0);

    1.5 $onehat

    在任意给定的时钟沿，表达式只有1个bit位为高。

    1.6 $onehat0

    在任意给定的时钟沿，表达式只有1个bit位为高，或者没有任何bit位为高。

    1.7 $isunknow

    检验表达式的任何bit位，是否是X或Z。

    1.8 $countones

    计算向量中为高的位的数量。

    a33a: assert 
            property(@(posedge clk) $onehot(state)); 
a33b: assert 
            property(@(posedge clk) $onehotO(state)); 
a33c: assert 
            property(@(posedge clk) $isunkiiovm(bus) ) ; 
a33d: assert 
            property(@(posedge clk)$countones(bus)> 1);

    二、操作符

    2.1 ##m 时钟延迟

    @(posdege clk) a ##2 b;，a高电平两个周期后b为高电平

    m可为0，表示a、b同时为高 ，可用于同一clk沿降时同时发生这两个事件

    2.2 ##[m:n]

    范围内即可

    2.3 [*m] 重复，连续

    @(posdege clk) a ##1 b[*2];，a高电平1个周期后，b高电平维持两个周期（连续两个clk采样到高）

    2.4 [*m:n]

    范围内即可

    2.5 [=m] 重复，不连续

    @(posdege clk) a |=> b [=2] ##1 c;，a ture1个周期后，b只要匹配到两个周期ture就可以了，下个周期后只要c为ture就行（即：c前有两个b成功就可）

    2.6 [=m:n]

    范围内即可

    2.7 [->m] GoTo重复，不连续

    @(posdege clk)a |=> b [=2] ##1 c;a ture 1个周期后，b只要匹配到两个周期ture就可以了，下个周期c必须为ture（即：c前必须有一次b成功挨着）

    2.8 [->m:n]

    范围内即可

    2.9 sig1 “throughout” seq1

    信号1在序列1检测过程中，必须一直为真

    例2、在整个检验过程中，start信号保持为低电平

    property p31; 
    ©{posedge clk) $fell(start) |-> 
                                    (!start) throughout 
                                    (##1 (!a&&!b) ##1 (c[->3]) ##1 (a&&b)); 
endproperty

//1、在clk上升沿检测到 start 下降沿后开始检测
//2、……
//3、……
//4、在整个检验过程中，start信号保持为低电平

    2.10 seq1 “within” seq2

    在一个序列中定义另一个序列，seq1必须在seq2的过程中发生，即seq2的开始匹配点必须在seq1之前，seq1的结束匹配点必须在seq2之前结束。

    2.11 seq1 ”and“ seq2

    当两个sequence都成功时，整个property才成功。

    2.12 seq1 “or” seq2

    只要其中一个sequence成功，整个property才成功

    2.13 seq1 “intersect” seq2

    和and类似，区别是：两个sequence必须在相同时刻开始，且结束于同一时刻。即seq1和seq2长度必须相等。

    2.14 first_match

    module assertion_example;
  bit clk, a, b, c;
  
  always #1 clk = ~clk;
  
  sequence s1;
    first_match(a ##[1:2] b);
  endsequence

  sequence s2;
       a ##[1:2] b;
  endsequence

  property p1;
    @(posedge clk) s1 |-> c;
  endproperty

  property p2;
    @(posedge clk) s2 |-> c;
  endproperty

  
 
a1: assert property(p1) $display("%t, success a1\n", $time); else $error("Assertion a1 failed!\n");
a2: assert property(p2) $display("%t, success a2\n", $time); else $error("Assertion a2 failed!\n");
    
  initial begin
    a = 0;
    b = 0;
    c = 1;
fork
    fork //a
      #2 a=1;
      #4 a=0;
      #8 a=1;
    join

    fork //b
      #2 b=1;
      #4 b=0;
      #8 b=1;

    join
join

    #20; $finish;
  end
endmodule

    a ##[1:2] b，如果没有|->或者|=>时，则匹配一次即可进入下一阶段或者断言成功，但是如果有|->或者|=>时，则必须保证所有情况都满足才能进入下一阶段，否则卡死。

    first_match的作用就是，使得只要出现一种满足情况即可进入下一阶段。

    如图可看到，加了first_match的p1，匹配了1个周期就开始匹配c了；而p2这完成了匹配1、2周期这两项，才进行c的匹配。

    2.15 not

    禁止属性，当property不成立时，断言为真。用于我们期望property永远为假的情况。

    2.16 .ended 和 .matched 结构

    区别：.end要求两个sequence拥有同一个clk， .matched允许两个sequence不同的clk。

    作用：匹配一个sequence的结束，如果sequence成功匹配并结束，那么.end/.matched为真。

    sequence s1; 
    @(posedge clk) a ##1 b; 
endsequence 
sequence s2; 
    @(posedge clk) c ##1 d; 
endsequence 

property p1; 
    s1 |=> s2; 
endproperty 
property p2; 
    s1.ended |-> ##2 s2.ended; 
endproperty  

al: assert property(p1); 
a2: assert property(p2);

    上面的例子，用两种方法都是检查了一个 a ##1 b ##1 c ##1 d 序列，

    如下图真值表，上面代码中的a1和a2 断言都匹配的是周期2-5。只是a1是匹配的开始，a2是匹配的s1和s2的结尾。

    2.17 expect 构造——（assert语句层面）

    expect相当于Verilog中的wait，只能用于procedural block中，有阻塞作用

    initial 
begin 
    @(posedge clk); 
    #2ns cpu_ready = 1'b1; 
    expect(@(posedge clk) ##[1:16] memory_ready == 1'b1) 
                $display("Hand shake successful\n"); 
        else begin 
                $display("Hand shake failed: exiting\n") 
                $finish(); 
        end 
//如果expect中memory_ready断言失败，后面这段被阻塞，不会执行
    for(i=0; i<64; i++) 
        begin 
        send_packet(); 
        $display("PACKET %0d sent\n", i); 
        end 
end

    2.18 disable iff

    1、只能用于property，不能用于sequence。

    2、只能声明在先行算子（antecedent condition）之前。

    property p34; 
    @(posedge clk) 
    disable iff (reset)//当reset信号为高时，后续断言不会执行 
    $rose{start) |=> a[=2] ##1 b[=2] ##1 !start 
endproperty 

a34: assert property(p34);
