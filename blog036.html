    一、模块的定义：SC_MODULE

    关键字：SC_MODULE

    解释：SC_MODULE 是 SystemC 库中定义的一个宏，使用它定义一个模块实际上是定义了一个新的 C++ 类。

    二、模块的端口：sc_in、sc_out、sc_inout

    端口使数据能够在模块间通过。

    端口分为三种类型：sc_in、sc_out、sc_inout。

    在端口定义时可指定数据类型：如 bool、int、short、char 等。

    举例，如下：

    SC_MODULE(fifo){
            sc_in_clk              clk;                 //clock
            sc_in<bool>            rst;                 //reset
            sc_in<bool>            wr_en;               //FIFO write enable
            sc_in<bool>            rd_en;               //FIFO read enable
            sc_in<int>             din;                 //data input, 32 bit bus
            sc_out<bool>           full, almost_full;
            sc_out<bool>           enpty, almost_empty;
            sc_out<unsigned short> data_count;          //FIFO data count
      sc_out<int>            dout;                //FIFO data out
}

    三、模块的信号：sc_signal

    模块之间通过信号将端口连接起来，信号是数据的载体，可以认为信号就是连线（类比于 Verilog 中的 wire）。

    关键字：sc_signal<T>

    举例，如下：

    如图36.2所示：operator_ctrl、 operand_ctrla、operand_ctrlb、a、b、c、result，为 7 个信号。

    注：信号和子模块的端口处于不同的设计层次，所以允许信号和端口同名，如：ALU 的 result 端口和信号 result。

    3.1 端口和信号的关系

    如图36.3所示，模块1和模块2分别有一个进程，假设这两个模块之间的信号为D；

    在T1时刻，Clk从0 ——> 1，恰好模块1中的进程也将信号 D 从0 ——> 1，但是模块1和模块2中的进程都在同一个上升沿被同时触发，它们工作在同一个时间步内，
    所以在T1时刻，模块2中的进程看到的 D 值是 0（旧值）。

    3.2 多端口和信号的快速声明

    sc_in<sc_logic> a[32];  //a[0]到a[31]，32个端口，方向为输入，数据类型为sc_logic
sc_signal<sc_logic> abus[16];  //abus[0]到abus[15]，16个信号

    3.3 隐式类型转换，助手函数read()和write()

    //如果定义了如下端口
sc_in<bool>  data_in;
sc_out<int>  data_out;

//data_out=data_in;
//在进程中直接赋值是非法的，因为两者类型不同
data_out.write(data_in.read());

    四、名字关联

    如图36.2所示，顶层模块 Top 的作用是：将各个子模块连接起来，子模块的端口是通过 Top 中定义的信号来关联的。

    如何将 Top 的信号和子模块的信号关联，名字关联和位置关联。

    例子：Top.h 文件如下

    #include "systemc.h"
#include "DataGenerator.h"  //For Gen_a and Gen_b
#include "Control.h"
#include "ALU.h"
#include "OperatorGenerator.h"
#include "DisplayResult.h"

SC_MODULE(Top) {
//指针定义
Control             *Ctrll;
DataGenerator       *Gen_a, *Gen_b;
OperatorGenerator   *OpGenl;
ALU                 *ALU1;
DisplayResult       *DispResultl;

//端口定义，Top层可以没有端口， 这里只有一个Clock端口
sc_in_clk   clk；

//信号定义
sc_signal<int>   a, b, result;
sc_sign<sc_unit<> >   c;
//知识点：sc_unit<>是SystemC中专有的数据类型，4表示数据只有4位，故可表示16中=种操作；
//在声明中，sc_sign<sc_unit<> >，中间有个额外的空格不能省略。
sc_sign<bool>   operator_ctrl, operand_ctrla, opcrand_ctrlb;

//Constructor1 位置关联（不推荐）
SC_CTOR(Top) {
    Ctrl1 = new Control("Ctrl1");  //创建一个新的对象指向它的指针
    (*Ctrl1) (clk, operand_ctrla, operand_ctrlb, operator_ctrl);//用对象的指针把信号关联到对象端口
//其它模块省略…………
}

//Constructor2 名字关联（推荐）
SC_CTOR(Top){
    Ctrl1 = new Control("Ctrl1");
    Ctrl1 -> clk (clk);
    Ctrl1 -> op  (operator_ctrl)
    Ctrl1 -> or_b(operand_ctrlb);
    Ctrl1 -> or_a(operand_ctrla);
//其它模块省略…………
}
}

    五、模块的构造函数：SC_CTOR

    关键字：SC_CTOR

    作用：1、创建模块内部数据结构，并把这些数据结构初始化为已知值。
    　　　2、初始化进程的类型，并创建进程的敏感列表。

    举例：

    //Constructor
SC_CTOR(asyn_fifo){
     SC_METHOD(write);  //进程write()对写时钟和复位信号敏感，工作在写时钟，异步复位。
        sensitive<<wr_clk<<rst;
         SC_METHOD(read);  //进程read()工作在读时钟，异步复位。
                sensitive<<rd_clk<<rst;
         SC_METHOD(synchronize_pointer);  //进程synchronize_pointer完成指针交换工作。
                sensitive<<wr_clk<<rd_clk<<rst<<ptr_wr<<ptr_rd;  //后两个是写指针和读指针
//定义为：sc_signal<int>  ptr_wr, ptr_rd; //write pointer and read pointer;

…………
for(i=0;i++;i<DEPTH) {mem[i]=0;}  //for循环把FIFO的存储空间初始化为0.
}

    注意：构造函数的名字必须与模块的名字相同

    六、SystemC时钟和时间模型

    //名称为clk1，表示20个时间单位，单位是ns，占空比是0.5(高电平持续0.5个周期)，
//在5ns时第一次变化（相位偏移5个单位）。
//ture：信号将从高电平开始，false：则信号从低电平开始。
sc_clock clk1("clk1", 20, SC_NS, 0.5, 5, true);
//名称为clk2，…………，在0ns时第一次变化
sc_clock clk2("clk2", 20, SC_NS, 0.5, 0, true);

//或者引入一个sc_time类型常量time1，通过time1初始化clk3；
sc_time time1(20， SC_NS);
sc_clock clk3("clk3", time1);

    //设置时间分辨率
sc_set_time_resolution(10, SC_PS);//分辨率为10ps

//1、时间分辨率必须是10的幂；
//2、……只能在仿真开始之前设置；
//3、……只能设置1次；
//4、必须在任何的非零的sc_time声明之前设置
