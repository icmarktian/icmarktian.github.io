    一、ERASE Resume功能描述

    • 重新开始指定LUN被Suspend的Erase操作。
    • 如图3，发送Command D7h经tWB后，R/B_n信号拉低，在tERES时间段内完成Erase Resume操作。
    • 在发送D7h前，Host应选择被Erase Suspend的LUN。
    • 如果Device未处于Erase Suspend状态，Erase Resume命令会被忽略。
    • 如果Multi-Plane Erase 操作被Suspend，则发送 D7h 会同时恢复所有的Erase操作。

    二、ERASE RESUME代码实现方案

    2.1、Command D7h

    代码如下，当IO口的数值为8h’D7会触发此条件语句。

    else if (Io_mirror [7 : 0] = = = 8'hD7) begin
      1、program cache 未完成时，报错ARRAY is still busy，直到状态变为E0h
      2、判断是否处于OTP_mode，OTP mode 不允许 ERASE Resume
      3、lastCmd = 8'hD7
      4、调用 clear_queued_planes、disable_rdStatus
      5、当检测到处于suspend 状态时，则传递待 RESUME 的 block地址
        else if (erase_susp_flag) begin
              for (eb_thisPlane=0; eb_thisPlane < NUM_PLANES; eb_thisPlane=eb_thisPlane+1) begin
           if (erase_susp_blk_addr[eb_thisPlane] ! = = {BLCK_BITS{1'bx}}) begin 
               queued_plane[eb_thisPlane] = 1;
               row_addr[eb_thisPlane][BLCK_BITS+PAGE_BITS-1:PAGE_BITS] = erase_susp_blk_addr[eb_thisPlane];
           end
         end
      6、erase_blk_en = ~ erase_blk_en;
      7、调用 clear_erase_suspend，清除与 Erase Suspend 相关的寄存器
      8、wait (erase_done);
     end
end

    2.2、Erase RESUME实现

    如2.1所示，第14行第6点，当IO收到D7h命令后同时满足RESUME条件，则通过“erase_blk_en = ~ erase_blk_en”产生擦除行为（如《7–[Design]ERASE BLOCK协议及代码实现-项目2-1》3.4描述）。
