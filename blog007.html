    一、项目功能描述

    本项目基于长江存储NAND Datasheet，实现了NAND behavior model的擦除功能（Erase Block），擦除挂起功能（Erase Suspend），擦除恢复功能（Erase Resume）。

    二、Erase Block功能描述

    • 以Block为单位对Array进行擦除。
    • 如图7.1，Command 60h后跟四个周期的Row地址，再发送Command D0h，经tWB时间后，R/B_n信号拉低，开始时长tBERS的擦除操作。
    • Host端通过监测R/B_n信号或者Status Register，判断Erase操作是否完成，
      SR[0]为0则表示Erase成功（SR[6]为1时，SR[0]才有效）。
    • 在Erase过程中，只可接受Read Status Command、Reset Command、Suspend Command。

    三、ERASE BLOCK代码实现方案

    3.1、命令输入always块

    如下所示，在We_n的上升沿会触发command input的always块。

    always @ (posedge We_n) begin : cLatch
          1、判断是否command_enable
          2、判断power up后第一个命令是否为reset命令，会产生error信息
          3、Command（60h）
          4、Command（D0h）
end

    command_enbale信号定义如下：

    assign command_enable = (Cle & ~Ale & ~Ce_n_vol & Re_n );

    3.2、Command 60h

    当IO口收到60h时，内部会进行如下操作：

    if (Io_mirror [7 : 0] = = = 8'h60) begin
        1、program cache 未完成时，60h会被忽略，直到状态变为E0h
        2、~Wp_n 和 OTP_mode 时均不允许 ERASE
        3、60h有效时，判断是否处于Multi-Plane ERASE，
           if lastCmd为60h，或D1h，则继续让lastCmd=60h
        else if (~multiplane_op_erase) begin
                              saw_cmnd_60h = 1'b1;
                              saw_cmnd_60h_clear = 1'b1;
                          end else begin
                              saw_cmnd_60h = 1'b1;
                              saw_cmnd_60h_clear = 1'b0;
                          end
        4、调用 disable_rdStatus，复位 status flag，让device返Read mode

end

    3.3、Command D0h

    当IO口收到D0h时，内部会进行如下操作：

    else if (Io_mirror [7 : 0] = = = 8'hD0) begin
      1、program cache 未完成时，D0h会被忽略，直到状态变为E0h
      2、判断是否 die_select，选择了才会往下执行
      3、if ((lastCmd = = = 8'h60) && row_valid && Wp_n) begin
          3.1、lastCmd = 8'hD0
          3.2、判断当前LUN是否有被 Erase Suspend 的blcok，若地址相同，将会Resume 被Suspend 的block
          3.3、调用clear_erase_suspend，清除和 Erase Suspend 相关的 reg
          3.4、erase_blk_en = ~ erase_blk_en，产生沿降，调用后续always 块开始erase操作
       end
    end

    3.4、Erase功能实现的always块

    第一个always块的作用是通过erase_blk_en和erase_blk_en_r的异或逻辑，使得一旦产生erase_blk_en = ~ erase_blk_en的变换时，确保erase_blk_pls都能产生上升沿逻辑，从而驱动Erase操作的always块。

    always @(*)
begin
    erase_blk_pls <= #tWB_delay ((erase_blk_en & ~erase_blk_en_r) | (~erase_blk_en & erase_blk_en_r));
    erase_blk_en_r <= #1 erase_blk_en;
end

    当erase_blk_pls产生上升沿的跳变时，会触发如下具有擦除逻辑的always块，从而完成Erase操作。

    always @(posedge erase_blk_pls) begin
      1、进入busy状态，设置SR[5、6]为00；
      2、调用 check_plane_addresses，检查排队的擦除块是否满足multi-plane寻址要求
      3、擦除逻辑
       if (eb_unlocked_erase) begin : unlocked_erase_block
            3.1、截取擦除的 block 地址 
             erase_block_addr = row_addr [eb_thisPlane] [BLCK_BITS + PAGE_BITS – 1 : PAGE_BITS];
            
        3.2、采用循环将array进行写1擦除
                      for (e=0; e<memory_used; e=e+1) begin : mem_loop
                          //check to see if existing used address location matches block being erased
                          if (memory_addr[e] [BLCK_BITS + PAGE_BITS - 1 : PAGE_BITS] = = = erase_block_addr) begin
                              mem_array[e] = {PAGE_SIZE{erase_data[0]}};
                              seq_page[erase_block_addr] = {PAGE_BITS{1'b0}};                 
                 mem_flag[e] = 0;  // 0: not programmed; 1: programmed.
                                                  end
                      end //mem_loop end
         end  // unlocked_erase_block
end
